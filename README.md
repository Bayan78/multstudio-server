# Cartoon Gen Backend

Необязательный сервер для МультСтудии, который:
1. **Хостит фото персонажа** (`/upload-ref`) — чтобы «Эталон из своего фото» работал
   (Pollinations kontext может ссылаться только на публичный URL).
2. **Проксирует генерацию** (`/generate`) с ретраями и (опц.) ключом Pollinations —
   быстрее, надёжнее, и картинки приходят с твоего домена → видео всегда собирается
   без CORS-проблем.

## Деплой на Railway
1. Залей содержимое папки в новый GitHub-репозиторий.
2. railway.app → **New Project → Deploy from GitHub repo**.
3. (Необязательно) во вкладке **Variables** добавь `POLLINATIONS_KEY`:
   - зарегистрируйся бесплатно на **auth.pollinations.ai**, получи ключ (тариф Seed);
   - лимит станет ~5 сек вместо 15 и уберётся вотермарка.
4. **Settings → Networking → Generate Domain** → получишь адрес
   `https://your-gen.up.railway.app`.
5. Проверь: открой адрес — `{ "ok": true, "key": "anonymous" | "set" }`.

## Подключение в приложении
В МультСтудии, карточка «Стиль и герой» → поле **«Сервер генерации»** вставь этот адрес.
Теперь:
- **📷 Эталон из своего фото** — загружаешь картинку персонажа, и все кадры рисуются по ней.
- Обычная генерация идёт через сервер (надёжнее + быстрее с ключом).

## Эндпоинты
- `GET /` — проверка.
- `POST /upload-ref` — multipart `image` → `{ url }`.
- `POST /generate` — JSON `{ prompt, seed, model:"flux"|"kontext", ref, width, height }` → картинка (bytes).

## Заметки
- Файлы-референсы лежат во временной папке и пропадают при перезапуске сервиса — это нормально
  для одной сессии создания мультика.
- Лимиты Pollinations всё равно действуют (это лимит аккаунта/IP). Ключ Seed их снижает.


## 🎬 Живое видео (движение персонажей)
Эндпоинт `POST /animate` превращает кадр в короткий movnign-клип (image-to-video)
через Pollinations Video (Seedance/Wan). Требует **POLLINATIONS_KEY с Pollen-кредитами**
(видео платное по кредитам; бесплатный ключ даёт очень мало). 
- Переменная `VIDEO_MODEL` (по умолчанию `seedance`) — можно сменить на `wan-fast`, `seedance-2.0` и т.д.
- Вход: JSON `{ frameUrl, prompt, duration, aspectRatio, model }` → отдаёт mp4.
- Клип генерится долго (десятки секунд); это нормально.
